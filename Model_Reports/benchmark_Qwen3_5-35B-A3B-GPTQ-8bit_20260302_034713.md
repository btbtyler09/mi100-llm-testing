## 6. Concurrency Scaling Analysis

The system exhibits near-linear scaling for aggregate throughput up to **Concurrency 16**.

1.  **Linear Scaling (c=2 to c=16):**
    *   As concurrency increases, the GPU utilization increases, maximizing the total output throughput (from 78 tok/s at c=2 to 454 tok/s at c=16).
    *   Latency (TPOT) remains relatively stable, increasing only from ~25ms to ~31ms. This indicates the system effectively hides latency behind batch processing.

2.  **Diminishing Returns (c=32):**
    *   **Throughput:** Increases to **582.67 tok/s** (only +28% over c=16).
    *   **Latency:** **TPOT jumps to 45.78 ms** (a ~48% increase from c=16).
    *   **Prefill:** Drops to 788.9 tok/s (vs 1745 tok/s at c=16).
    *   **Conclusion:** At c=32, the system likely hits memory bandwidth saturation or PCIe/Infinity Fabric contention limits. The cost of managing 32 concurrent request queues outweighs the compute gains.
