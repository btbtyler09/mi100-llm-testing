This report evaluates the inference performance of the **Qwen3.5-35B-A3B** model (4-bit quantized) deployed across **4× AMD Instinct MI100** GPUs. The architecture excels in batch processing scenarios but requires careful concurrency management for interactive low-latency applications.

2.  **Decode Instability:** TPOT increases from 23ms (single user) to 70ms (c=32). This 3x latency increase on the generation phase suggests that the MI100 compute units are heavily contended, likely due to the MoE activation routing overhead on older hardware.

## 7. Key Observations

1.  **MoE Efficiency:** The Qwen3.5-A3B MoE structure allows for reasonable latency on MI100. A dense 35B model would likely struggle more on this hardware. The 4-bit quantization is essential to fit the KV cache within the 64GB (4x16GB) VRAM constraint.
2.  **MI100 Memory Bandwidth:** The MI100 (HBM2) architecture shows signs of memory saturation at high concurrency. The drastic drop in Prefill speed at c=32 confirms that the system is bandwidth-bound rather than compute-bound for the prefill phase.
3.  **Latency Variance:** In the Single User test, the TTFT p99 (8907ms) is significantly higher than the mean (766ms). This suggests occasional micro-stalls or garbage collection pauses in the inference engine, which should be investigated for production stability.
4.  **Long Context Strength:** Despite being an older generation GPU, the system handles 16K context with a high prefill throughput (4000 tok/s), outperforming the single-user baseline. This is likely due to the amortization of communication overhead across the 4 cards during larger batch processing.

### For Batch Processing / RAG / Summarization
*   **Recommended Concurrency:** **16 to 24**.
*   **Reasoning:** This range captures the peak throughput efficiency (~300 tok/s) before the latency penalty becomes too severe for batch jobs.
*   **Strategy:** Use a timeout mechanism to drop requests that exceed 1-second TTFT.

### For High-Volume Throughput (Non-Interactive)
*   **Recommended Concurrency:** **32+**.
*   **Reasoning:** While TTFT is poor (1.8s), the total system throughput peaks at **1912 tok/s**.
*   **Caveat:** Ensure the application layer can handle asynchronous responses or long polling, as this configuration is unsuitable for streaming chat.

### Hardware Upgrade Path
*   If interactive latency is critical (<200ms TTFT), consider upgrading to **AMD Instinct MI250/MI300** or **NVIDIA H100/A100**. The MI100's HBM2 bandwidth is the limiting factor for scaling beyond 16 concurrent requests.
*   If VRAM capacity is the bottleneck (35B model on 4x16GB), moving to **32GB HBM2e** or **HBM3** cards would allow for larger batch sizes without sacrificing throughput.
